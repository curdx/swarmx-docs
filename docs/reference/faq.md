# 常见问题

## 至少要装哪个引擎？

**claude 是最低要求**，装好并登录它就能开始。想要跨引擎协作再装 codex；想用研究委员会与融合竞赛的多模型，装 zulu 并配置 License。见[安装与登录引擎](/guide/install-engines)。

## 引擎显示「已安装」但用不了？

「已安装」只代表命令能找到，不代表能跑。到「设置 → 插件」点**检测可用性**，它会真实冷启动每个引擎并给出「可用 / 未验证 / 需登录 / 无法启动」的判定。多数「用不了」是**未登录**——按卡片上的登录命令登录即可（如 claude 在终端运行一次 `claude` 登录、`codex login`）。

## codex 的自动唤醒不生效？

codex 的回合末自动唤醒需要 **0.132 及以上**。更低版本引擎照常运行，但自动唤醒处于休眠状态。升级 codex 即可。见[引擎参考](/reference/engines)。

## 研究委员会/融合竞赛里没有模型可选？

这两者的多模型由 zulu 提供，需要配置 Comate License。到「设置 → 插件」填入 License，模型列表就会出现。见[安装 Zulu 与配置 License](/guide/zulu-setup)。

## 浏览器里创建工作空间时选不了文件夹？

桌面版可用原生对话框选文件夹；**浏览器调试模式下需手动粘贴项目目录的绝对路径**。路径框会实时校验是否存在、可访问。

## 新方向没有被隔离？

自动隔离依赖 git。**非 git 项目无法隔离**，多条方向会共用同一目录、可能互相覆盖（界面会提示）。把项目变成 git 仓库即可获得每方向独立的工作副本。见[工作方向与隔离](/guide/directions-and-isolation)。

## 用量页的成本准吗？

是**估算值**。因为 claude、codex 走终端而非 API，服务端从会话记录里抓取 token 数并按内置价目表估价，未知模型只统计 token。把它当量级参考，而非账单。

## 支持 Windows 吗？

swarmx 面向 **macOS 与 Linux**。发布流程也构建 Windows 产物，但 **Windows 为实验性、未经完整验证**——进程回收在非 Unix 上不完整。建议在 macOS 或 Linux 上使用。

## 顶部出现「后端离线」横幅？

说明本地服务端中断了。桌面版可用横幅上的重试/重启恢复；源码运行则确认 `swarmx-server` 仍在运行（见[快速开始](/guide/quickstart)）。

## swarmx 会上传我的代码或凭证吗？

不会。服务端只监听本地回环地址，且从不读取或存储任何登录凭证——只复用你 CLI 已有的登录。详见[数据与隐私](/reference/data-and-privacy)。
