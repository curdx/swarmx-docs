# 融合竞赛

融合竞赛把**同一个需求**丢给几个模型，各自在隔离的 git worktree 里独立实现；可以挂一条
**客观检查命令**（比如 `pytest`、`cargo test`）当硬门禁，judge 再综合各家最优、合并回主线。

## 一键全自动

新手只要填一句需求、勾上「全自动」、点一下「一键开赛」，后面全自动：

1. 服务端自动挑选可用模型组成 panel（有两个以上不同 CLI 就跨引擎对赛，否则用一把 zulu
   license 跑多个模型）。
2. 每个模型在自己隔离的 worktree 里实现需求。
3. 选手都提交后，自动进入综合裁判阶段。
4. judge 综合出最优版，合并回主线。

全程零手动。裁决由**服务端看门狗**兜底：无论模型是否规规矩矩收尾，批次都不会静默卡死——
该合并的合并，该退回人工的退回人工。

## 手动模式

关掉「全自动」，就是经典流程：自己命名 2–4 个选手、驱动它们实现，然后点「进入评判 / 自动
评判 / 综合最优」，最后选一个赢家合并。适合你想全程掌控每一步的时候。

## 客观检查（可选但推荐）

给一条 check 命令（如 `python3 check.py`、`cargo test`），swarmx 会在每个选手的 worktree 里
真实跑一遍。这是比「读 diff」更硬的证据：任何客观检查失败的选手，一律不当选——哪怕它的
代码看起来最优雅。

## 前置

多模型由 Comate Zulu 驱动（跨引擎模式则用你已登录的 claude / codex 等）。配置见
[装 Zulu、配 license](/guide/zulu-setup)。
