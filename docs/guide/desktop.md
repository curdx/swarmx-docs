# 桌面版

桌面版把 swarmx 打包成一个原生应用：**下载、安装、打开即可使用，全程无需命令行**。它适合只想用、不想配环境的用户。

## 开箱即用

桌面版将服务端及其配套二进制一并内嵌，所需的内置资源（编排器、角色、引擎清单等）在编译期打入应用。因此即便应用运行时的工作目录并非某个项目仓库、也没有任何环境变量，它依然能正常启动——新用户的完整路径就是「下载 → 安装 → 打开 → 立刻能用」，中间不需要敲任何命令。

打开后，你仍需备好至少一个已登录的编码 CLI（见[安装与登录引擎](/guide/install-engines)）；引擎可以在应用内的「设置 → 插件」里一键安装并检测可用性。

## 平台支持

swarmx 面向 **macOS 与 Linux**。进程回收依赖 Unix 的进程组信号来确保被拉起的 CLI 及其子进程被干净地结束，数据目录也默认基于用户主目录。发布流程同时构建 Windows 产物，但 **Windows 目前为实验性、未经完整验证**——在非 Unix 平台上，孙进程可能无法被可靠回收。建议在 macOS 或 Linux 上使用。

## 自动更新

桌面版内置更新检查：发现新版本时，「设置 → 关于」会提示，可在应用内下载并安装。

## 从源码构建安装包

如果你想自己出安装包，在 `web/` 目录下：

```bash
npm run sidecar:release   # 构建 release 服务端并复制为内嵌二进制
npm run tauri:build       # 生成真实安装包（.app / .dmg 等）
```

开发调试用 `npm run tauri:dev`。注意：开发构建下应用不会自动拉起服务端，需自行运行 `cargo run -p swarmx-server`（见[快速开始](/guide/quickstart)）。
