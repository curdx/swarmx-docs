# 蜂群协作

蜂群协作是 swarmx 的默认模式。你只与一个常驻的[编排器](/guide/orchestrator-and-roles)（队长）交互，由它根据任务规模伸缩团队：小事自己做，大事拆开派几个 worker 分头干。无需预先设计流程，成员按任务需要在运行时派发。

![用自然语言与编排器对话](/assets/screenshot-orchestrator-chat.png)

## 如何开始

1. [创建工作空间](/dashboard/create-workspace)，指向一个真实的项目目录。编排器会先扫描项目、写好台账，再向你打招呼。
2. 用自然语言把目标告诉它——一个需求、一个 bug、一次改造都行。
3. 它会判断是直接完成，还是拆分派发。你在[聊天视图](/dashboard/workspace-views)里看到它的计划、派出的成员与进展。

## 成员如何协作

成员之间不共享上下文，而是通过两种显式载体协作（详见[黑板与收件箱](/guide/blackboard-and-inbox)）：

- **共享收件箱**——agent 以 id 互相寻址，消息在对方下一个回合边界送达，无需轮询。
- **共享黑板**——支持全文检索与版本历史的键值存储，任意写入即时推送给订阅者。
- **即时唤醒**——某个黑板键一被写入，所有等待它的 agent 在同一时刻被唤醒，包括此前已停下的（详见[唤醒机制](/guide/wake-mechanism)）。

成员的产出通过**类型化交接**衔接：谁产出、谁消费、在什么信号上唤醒，都是显式且可校验的，不靠约定俗成的文件名。

## 跨引擎的团队

编排器与 worker 可以是不同引擎。一个常见组合是 claude 编排器搭配 codex worker——各取所长。每个引擎都以统一的 agent 形态出现在仪表盘里，差异见[引擎参考](/reference/engines)。

## 可视化的协作过程

蜂群协作的每一步都看得见：

- **[聊天](/dashboard/workspace-views)**——与队长对话，顶部固定一张实时更新的计划清单，右侧是成员栏。
- **[协作图](/dashboard/workspace-views)**——把成员之间的依赖与等待关系画成实时图谱。
- **[工作台账](/dashboard/workspace-views)**——展示队长维护的任务台账、进展与各成员的最新动态。
- **[录像](/dashboard/workspace-views)**——每个成员的终端会话都被录下，可回放复盘。

![蜂群的实时依赖图](/assets/screenshot-dag.png)

## 一个例子

请编排器「创建两个文件，各由一个 worker 负责」，你会看到它上线、派出两个 worker（可以是不同引擎）、各自在隔离的工作副本中完成、提交并汇报，队长再核验后向你汇总。整个过程中，计划清单、协作图与成员栏同步反映真实状态。

## 并行推进多条线

同一个工作空间里可以开多条[工作方向](/guide/directions-and-isolation)并行推进，互不干扰——在 git 项目里每条方向自动获得独立的工作副本与分支，完成后可合并回主线。
