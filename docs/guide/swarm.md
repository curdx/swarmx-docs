# 蜂群协作

蜂群协作是 swarmx 的默认模式。你只与一个常驻的**编排器**（orchestrator）交互，由它根据任务规模伸缩团队。

向编排器描述目标后，它会自行判断是直接完成，还是拆分为若干 worker 分头执行——无需预先设计流程，agent 按任务需要在运行时派发。

![用自然语言与编排器对话](/assets/screenshot-orchestrator-chat.png)

## 成员如何协作

- **共享收件箱**：agent 之间以 id 互相寻址，消息在对方下一个回合边界送达，无需轮询。
- **共享黑板**：一块支持全文检索与版本历史的键值存储，任意写入都会即时推送给订阅者。
- **即时唤醒**：当某个黑板键被写入时，所有等待该键的 agent 在同一时刻被唤醒——包括此前已停下的 agent，无需轮询，也不会死锁。

## 可视化的协作过程

协作视图将成员之间的依赖与等待关系绘制为实时图谱；台账视图展示任务进度与共享区状态。

![蜂群的实时依赖图](/assets/screenshot-dag.png)

以一个简单任务为例：请编排器「创建两个文件，各由一个 worker 负责」。你会看到编排器上线、派出两个 worker（二者可以是不同引擎，例如 claude 编排器搭配 codex worker）、各自完成后向你汇报。
