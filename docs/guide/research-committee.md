# 研究委员会

重要决策不应只依赖单一模型。研究委员会让多个模型并行回答同一问题，再由评审模型将各家答案归纳为**共识、分歧、独特观点与盲区**——是对比，而非投票——最终由综合模型给出一份定稿。技术选型、方案评审、竞品分析，或为高风险决策寻求反方意见，都比反复追问单一模型更可靠。

它是一种「问答/研究」型的多模型协作，与[融合竞赛](/guide/fusion)的「代码竞赛」不同：这里产出的是结论，不是代码。多模型能力由 [Comate Zulu](/guide/zulu-setup) 提供——一份 License 即可调用十余个模型。

![研究委员会：多模型并行作答、结构化对比、综合定稿](/assets/screenshot-consult.png)

## 使用方式

1. 打开工作空间的「研究委员会」标签。
2. 输入一个值得多模型会诊的问题。
3. 选择 **2 至 8 个模型**组成参与阵容（默认给一组跨厂商组合，缺失时回退到前几个可用模型）。
4. 点击「开始会诊」，等待约 1 至 2 分钟。

## 背后发生了什么

1. **并行作答**——每个入选模型独立回答同一问题，并行执行。只要有一个模型答出，流程就继续；仅当全部失败才报错。
2. **结构化对比**——一个评审模型把各家答案归纳成四个维度：**共识**（多数认同的点）、**分歧**（彼此矛盾之处）、**独特观点**（只有某一家提到的点）、**盲区**（都没顾及的重要方面）。它被明确要求做对比、不投票、不拼接。
3. **综合定稿**——一个综合模型基于上述对比写出一份最终答案：以共识为主干，对分歧给出更优的一方及理由，纳入独特观点，补上盲区——是重新落笔，不是简单拼接。

## 结果包含

- **综合定稿**——综合模型给出的最终答案。
- **结构化分析**——共识 / 分歧 / 独特观点 / 盲区四栏，各带条目数。
- **原始答案**——每个模型的完整回答与耗时，可展开逐一比对；失败的模型也会标注。

## 成本

单次会诊约相当于「模型数 + 2」次单模型调用（N 个参与模型，加一次归纳、一次综合）。这是一项面向高价值决策的能力，而非日常默认路径——当一个更可靠的结论值得数倍成本时使用。
