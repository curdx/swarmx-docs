# 研究委员会

重要决策不该只问一个模型。研究委员会让若干模型并行回答同一个问题，一个 judge 把它们的
答案拆成**共识、分歧、独特点、盲区**来对比——是对比，不是投票——再由外层模型综合出一份
定稿。技术选型、方案评审、给一个高风险决定找反方意见，都比反复追问单个模型强。

![研究委员会：多模型并行答题、结构化对比、综合定稿](/assets/screenshot-consult.png)

## 怎么用

1. 打开工作空间的「研究委员会」标签。
2. 填一个值得多模型会诊的问题。
3. 选 2–8 个模型（默认给一个跨厂商的组合）。
4. 点「开始会诊」，等 1–2 分钟。

结果分三块：

- **综合定稿**：外层模型基于对比写出的最终答案。
- **结构化分析**：共识 / 矛盾 / 独特洞察 / 盲区四栏。
- **原始答案**：每个模型各自的回答，可展开逐一比对。

## 成本

大致是 `(模型数 + 2)` 次单模型调用（N 个 panel + 一次 judge + 一次综合）。这是一个高价值
任务按钮，不是日常默认路径——值得多花几倍成本换一个更靠谱的结论时才用它。
