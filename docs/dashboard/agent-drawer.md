# Agent 抽屉

点击任意成员（在聊天成员栏、协作图或派发卡上），会从右侧滑出一个抽屉，让你深入这一个 agent。抽屉的地址可分享，也会记住你停留的标签页。

## 头部与操作

头部显示头像、角色、引擎、id，以及状态：**就绪 / 启动中 / 已结束 / 出错·无法工作**，外加已运行时长。

操作按钮：

- **唤醒**——让停下的 agent 续跑一个回合。
- **暂停 / 恢复**——暂停会中断当前动作并挂起自动唤醒，恢复则重新接入（有二次确认）。

## 五个标签

| 标签 | 内容 |
|---|---|
| **终端** | 该 agent 的实时终端画面。已退出的 agent 会提示终端不可用；无终端界面的引擎（如通过控制接口运行的 opencode、无界面的 reasonix）会引导你去看「活动」。 |
| **活动** | 分步的实时动作流（每回合的工具/系统步骤），已结束的回合折叠为一行摘要；历史从缓冲区回填，不会丢。 |
| **录像** | 这个 agent 的会话录像，可跳转到全屏播放器或下载。 |
| **消息** | 收发的消息列表，过滤到与这个 agent 相关的往来。 |
| **上下文** | 这个 agent 写过的黑板键，点击可跳到对应方向的台账并定位该键。 |

不同引擎的默认标签不同：已结束的 agent 默认开在「录像」（最适合复盘），无界面的引擎默认开在「活动」，在线的默认开在「终端」。

## 状态条

抽屉底部有三个实时指标：

- **SPAWN**——启动至今的时长。
- **PTY**——终端是否在线。
- **HOOK**——自动唤醒钩子是否激活。
