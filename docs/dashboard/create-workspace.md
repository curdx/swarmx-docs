# 创建工作空间

一个工作空间 = 一个项目 + 一支 AI 团队。点击工作空间侧栏顶部的**新建工作空间**，进入一个两步向导。

## 第一步：命名与队长引擎

- **名字**——给这个工作空间起个清晰的名字。
- **标识色**——从五种颜色里选一个，便于在侧栏区分多个工作空间。
- **队长引擎**——选择由哪个引擎担任编排器（默认 Claude，或任何已安装的引擎）。未安装的引擎会标注「未安装」且不可选。

## 第二步：选择主项目

- **主项目**——填入项目目录的绝对路径。界面会实时校验路径是否存在、可访问，并对「目录不存在 / 不是目录 / 无权限」给出友好提示。桌面版可用原生对话框选择文件夹；浏览器调试模式下需手动粘贴路径。

### 高级：源码上下文

展开「高级源码上下文」可挂载项目**之外**的目录作为参考，每个标注角色——**对等项目 / 依赖源码 / 工具项目**；依赖与工具还能挂到某个父项目之下，形成一棵依赖树。挂载后，编排器会直接读到这些源码，而不是凭空猜测。概念详见[工作方向与隔离](/guide/directions-and-isolation)。

## 创建与扫描

点击「开始」后，swarmx 会创建工作空间、准备好各个根目录，然后拉起编排器做一次项目扫描：它进入目录、识别技术栈、写下一份项目摘要，并向你打招呼。这一步通常 20–40 秒，界面会显示扫描进度，你也可以点「直接进入房间」提前进入、在聊天里等它上线。

进入房间后，就可以用自然语言把目标交给编排器了——参见[蜂群协作](/guide/swarm)与[工作空间视图](/dashboard/workspace-views)。
