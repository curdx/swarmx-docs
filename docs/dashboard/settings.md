# 设置

「设置」分为若干区（可用 `⌘1`–`⌘7` 跳转）。偏好类设置存在本机浏览器，模型与插件等配给存在服务端。

## 通用

界面语言；启动行为（启动时打开主面板）；新消息桌面通知；以及「失败时同时关闭其余 agent」——当一个成员失败时是否连带停掉同批其余成员。

## 外观

主题：浅色 / 深色 / 跟随系统。

## 快捷键

快捷键参考（目前为只读列表）。

## 模型

按引擎配给模型：为每个 CLI 指定使用哪个模型与思考强度。opus/sonnet/haiku 层级是 Claude 的原生说法，其余引擎取各自的默认模型 id。有未保存改动的提醒。

## 插件

引擎与多模型都在这里管理。

- **Comate License** 卡片——填入并保存 zulu 用的 License（仅存本机）。配置后列出这份 License 可用的全部模型。详见[安装 Zulu 与配置 License](/guide/zulu-setup)。
- **检测可用性**——真实冷启动每个引擎，确认它不只是「命令能找到」，而是确实能跑；每个引擎给出「可用 / 未验证 / 需登录 / 无法启动」的判定。
- **引擎列表**——每个引擎一张卡片，显示安装状态、二进制与版本。未安装的可**一键安装**（实时日志），并提供验证、登录命令与官方文档链接。完整说明见[安装与登录引擎](/guide/install-engines)。

## 隐私与安全

工具调用审批按产品设计恒为「放行」（无人工逐次确认）。可**导出**或**清空**存在本机浏览器里的本地数据。更完整的隐私说明见[数据与隐私](/reference/data-and-privacy)。
