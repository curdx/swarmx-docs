# 工具页

左侧导航栏里，「对话」之外的几项是围绕工作空间的工具页。每个都默认指向你最近使用的工作空间。

## 目标

任务之上的持久层：为工作空间记录**目标**，附**验收标准**、**状态**（活跃/暂停/阻塞/完成/归档）、可选的 **Token 预算**，以及作用的**方向**（全部或主方向）。每个目标可累积**证据**条目，分为记录、证明、阻塞三类。适合长任务，防止 agent 跑偏。

## 任务

一块看板，**每个成员即一个任务**。列包括：待分诊、待办、就绪、进行中、阻塞、完成、归档。状态通常从 agent 的生命周期自动推导，你也可以对某张卡手动覆盖（阻塞/完成/归档/复位）。该页通过轮询保持刷新。

## 定时

按 **5 段式 cron** 定时把一段提示投递给某个工作空间的队长。提供常用预设（每小时/每天/工作日/每周/每月），实时显示人类可读的描述与服务端计算的**下次运行**时间；可**立即运行**、启用/停用、就地编辑、删除。时间以浏览器本地时区显示。使用前需至少有一个工作空间作为投递目标。

## 用量

token 用量与**估算**成本。由于 claude、codex 走终端而非 API，服务端从每个 worker 的会话记录里抓取每回合的 token 数，再按内置价目表估价（未知模型只统计 token）。页面包含总览卡片（总成本/总 token/输入输出/缓存/峰值上下文）、按天趋势，以及**按模型**和**按 agent** 的拆分，还可查看价目表。成本为估算值。

## 文件

对工作空间各根目录的**只读**本地文件浏览器；有一个开关可解除限制、浏览整个文件系统（附警告）。支持 Markdown 与代码的语法高亮预览；二进制或过大的文件只标注、不展开。

## 终端

在浏览器里开一个交互式 shell（按工作空间，会话在页面间保持）。首次使用需点「连接终端」确认，断开时会提示重连。
