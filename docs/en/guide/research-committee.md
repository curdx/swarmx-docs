# Research committee

Important decisions should not rest on a single model. The research committee has multiple models answer the same question in parallel, then a reviewer model distills their answers into **consensus, disagreements, unique insights, and blind spots** — a comparison, not a vote — and finally a synthesis model delivers the conclusion. For technology selection, design review, or seeking a dissenting opinion on a high-risk decision, this is more reliable than repeatedly pressing a single model.

![Research committee: parallel answers from multiple models, structured comparison, synthesized conclusion](/assets/screenshot-consult.png)

## How to use it

1. Open the **Research committee** tab in a workspace.
2. Enter a question worth a multi-model consultation.
3. Select 2 to 8 models (a cross-vendor default set is provided).
4. Click **Start consultation** and wait one to two minutes.

The result has three parts:

- **Synthesized conclusion** — the final answer the synthesis model derives from the comparison.
- **Structured analysis** — the four dimensions of consensus, disagreements, unique insights, and blind spots.
- **Raw answers** — each model's complete reply, expandable for side-by-side comparison.

## Cost

A single consultation costs roughly "number of models + 2" single-model calls (N participating models, plus one distillation and one synthesis). This is a capability for high-value decisions, not a default everyday path — use it when a more reliable conclusion is worth several times the cost.

## Prerequisite

The research committee's multi-model capability is provided by Comate Zulu. Complete [Install Zulu and configure a license](/en/guide/zulu-setup) first — a single license gives access to more than a dozen models.
